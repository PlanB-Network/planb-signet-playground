# Squad 4 — Website & Global Presentation

> The front door of the playground. If a stranger lands here, they should understand the project and start playing within 5 minutes.

## Goal

Make Plan B Signet Playground understandable and usable by any external educator or student in minutes.

## Deliverables

- [ ] Public web page (single HTML file or small static site) covering:
  - [ ] What Plan B Signet is and why it exists
  - [ ] How to connect (wallet config, network params)
  - [ ] How to play (faucet → quiz)
  - [ ] Links to faucet, quiz, educator toolbox
- [ ] Project name + visual identity (logo, color palette, tone of voice)
- [ ] Communication plan: where to announce, which meetups, social channels
- [ ] **Onboarding tutorial for external educators** ("vibecoders onboarding")

## Stack

- HTML/CSS, or a lightweight static site generator (Astro, Eleventy, Hugo).
- **No heavy framework** — biz students should be able to edit copy without setting up a dev environment.

## Dependencies

- Squad 1 & 2: stable URLs to link to
- Squad 3: network connection params for the "how to connect" section
- QA: link-check + readability pass before publishing

1. Open an issue tagged `squad/website`.
2. Drafts go in `website/drafts/` until reviewed.
